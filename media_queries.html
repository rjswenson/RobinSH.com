Your website for all

By: Robin Swenson-Healey


Same website, different look

So you've made a beautiful website. It looks good locally on your computer but how does it look on anyone elses'?

Today more than ever people are viewing web pages on a plethora of devices. From new 4K monitors(3840 pixels Ã— 2160 lines) to Android Phones(767px or less) it is important that you don't lose viewers because of your broken design.

Getting Started

You'll need some basic tools to test your application design.

  • Chrome with Dev-Tools
  • Mobile emulation set up
  • Optionally: A Mobile/RWD Tester(extension)

With these we can test our current design and see where the problems lie.

Do's and Don'ts

When designing a page, sometimes we get caught up in making it look just right. This is an excellent habit, as long as you aren't modifying static positioning. A sure fire way to ruin your website for mobile devices include:

  • Static widths and heights for menus, and columns. Limit CSS which calls dimensions by pixel length(e.g. Width: 92% is a lot better than Width: 600px)
  • Text should be sized depending on the device, not pixel height(e.g. font-size: .8em not font-size: 12px)
  • Overall, limit fixed, static design(e.g. don't use position: fixed or absolute unless it really makes sense)

Media Queries to the Rescue

Writing your design using em and percentages will definitely pave the way for a good design. What really make a website pop though are 'Media Queries'

What is a Media Query?

CSS3 has introduced a new way of design with media queries. These CSS snippets allow a designer to modify a view based upon either the device itself or a particular size of screen.

NOTE: These are not currently supported by IE 8

Using Media Queries

CSS syntax for these are fairly straightforward.


  @media only screen and (max-device-width: 480px) {
    #navbar {
      width: 300px;
    }
    #content {
      float: none;
      width: 100%
    }
  }

This code changes our navbar and content to view differently, but only when the device is both a screen and under 480px in width.

There are many different applications for these queries.They can even be used with orientation.


  @media (min-width: 700px) and (orientation: landscape) {
  ....
  }

To find out all the options for media queries check out Mozilla's explanation and also these Media query snippets created for particular devices.

The onus is on you, the developer, to make your site work with different devices. Don't lose users simply because they can't use your static site.
